## üß† Why Use Redis in ASP.NET Core?

Redis is an in-memory, key-value data store ‚Äî ideal for:

| Use Case               | Why Redis?                                                   |
| ---------------------- | ------------------------------------------------------------ |
| üîÅ Distributed Caching | Share cache across multiple API servers                      |
| üîê Session Storage     | Persist session across multiple backend nodes                |
| üöÄ Performance Boost   | Store frequently accessed or expensive data (e.g., DB calls) |
| üïπ Pub/Sub or Locking   | Use Redis for messaging or distributed locking               |

## üß™ 4. Use Redis for Caching in Services

```csharp
public class ProductCacheService
{
    private readonly IDistributedCache _cache;

    public ProductCacheService(IDistributedCache cache)
    {
        _cache = cache;
    }

    public async Task SetProductAsync(string key, string jsonProduct)
    {
        var options = new DistributedCacheEntryOptions
        {
            AbsoluteExpirationRelativeToNow = TimeSpan.FromMinutes(10)
        };
        await _cache.SetStringAsync(key, jsonProduct, options);
    }

    public async Task<string?> GetProductAsync(string key)
    {
        return await _cache.GetStringAsync(key);
    }
}
```


## üî¨ 6. Use StackExchange.Redis for Low-Level Control

```csharp
public class RedisAdvancedService
{
    private readonly IConnectionMultiplexer _redis;

    public RedisAdvancedService(IConnectionMultiplexer redis)
    {
        _redis = redis;
    }

    public async Task StoreAsync(string key, string value)
    {
        var db = _redis.GetDatabase();
        await db.StringSetAsync(key, value);
    }

    public async Task<string?> RetrieveAsync(string key)
    {
        var db = _redis.GetDatabase();
        return await db.StringGetAsync(key);
    }
}
```

This allows:

- ‚úÖ Full key management
- ‚úÖ TTL configuration
- ‚úÖ Pub/Sub, Streams, Sorted Sets, Hashes, etc.

## üß† Example: Real Use Case ‚Äì Caching Expensive DB Result

```csharp
public async Task<List<Product>> GetCachedProductsAsync()
{
    string cacheKey = "product:list";
    string? cached = await _cache.GetStringAsync(cacheKey);

    if (!string.IsNullOrEmpty(cached))
        return JsonSerializer.Deserialize<List<Product>>(cached)!;

    // Expensive DB call
    var products = await _dbContext.Products.ToListAsync();

    await _cache.SetStringAsync(
        cacheKey,
        JsonSerializer.Serialize(products),
        new DistributedCacheEntryOptions
        {
            AbsoluteExpirationRelativeToNow = TimeSpan.FromMinutes(15)
        });

    return products;
}
```

## ‚úÖ Tips for ASP.NET Core + Redis Integration

| Best Practice                  | Why?                                           |
| ------------------------------ | ---------------------------------------------- |
| üîë Prefix keys                 | Avoid conflicts across app domains             |
| ‚è≥ Always set TTL              | Avoid memory bloat                             |
| üßº Handle serialization errors | Graceful fallbacks if cache fails              |
| üìà Monitor latency             | Redis client logs timeouts, retry strategies   |
| üö´ Never cache sensitive data  | Use MemoryCache or secure vaults for such data |
| üíî Fallback logic              | Always handle Redis unavailability gracefully  |

---

## üîÅ Bonus: Cache Invalidation Pattern (Write-Through)

On DB update:

```csharp
await _dbContext.SaveChangesAsync();
await _cache.RemoveAsync("product:list");
```

So the next read fetches fresh data and repopulates the cache.
